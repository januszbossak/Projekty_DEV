# Organizacja działu DEV

## Przegląd

Ten katalog zawiera dokumentację ustaleń organizacyjnych działu R&D – procesów, zasad, narzędzi, struktury zespołu.

**WAŻNE:** Treści z tego katalogu **NIE trafiają** do dokumentacji projektowej (`projekty/`). To osobna baza wiedzy o tym **jak pracujemy**, nie **co budujemy**.

---

## Kategorie tematów

### 1. Procesy i workflow
- Code review
- Branching strategy
- CI/CD
- Release process

### 2. Narzędzia i infrastruktura
- Azure DevOps
- Git
- Środowiska DEV/TEST/PROD
- Dostępy

### 3. Zespół i komunikacja
- Struktura zespołu
- Role i odpowiedzialności
- Spotkania cykliczne

### 4. Standardy i dokumentacja
- Konwencje nazewnictwa
- Standardy kodowania
- Wiki

### 5. HR i organizacja
- Urlopy, zastępstwa
- Onboarding
- Szkolenia

---

## Źródła notatek

Ustalenia organizacyjne mogą pochodzić z:
- **Dedykowanych spotkań organizacyjnych**
- **Fragmentów innych spotkań** (Rada Architektów, Planowanie) – wyodrębnionych do osobnych notatek

W przypadku wyodrębnienia, nazwa pliku zawiera źródło:
```
Ustalenia organizacyjne 2025-11-20 (z Rady Architektów).md
```
