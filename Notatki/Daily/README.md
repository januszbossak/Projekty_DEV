# Daily

Notatki ze spotkań Daily zespołu R&D AMODIT.

## Charakterystyka Daily

Daily to codzienne, krótkie spotkania operacyjne mające na celu:
- Synchronizację pracy zespołu
- Identyfikację blokerów
- Omówienie nowych zgłoszeń do backlogu
- Koordynację działań

## Struktura Daily

Daily często składa się z dwóch części:

### Część 1: Status update (wszyscy uczestnicy)
- Każdy uczestnik mówi co robi
- Czy są blokery
- Co planuje zrobić dalej

### Część 2: Omówienie backlogu (węższe grono)
- Nowe zgłoszenia napływające do backlogu
- Omówienie priorytetów
- Decyzje o przypisaniu zadań

### Część 3: Tematy organizacyjne (opcjonalnie)
- Urlopy, zastępstwa
- Spotkania, zmiany w harmonogramie
- Inne kwestie organizacyjne

## Format notatek

**Nazwa pliku:** `YYYY-MM-DD Daily.md`

**Struktura notatki:**
1. Status update od uczestników
2. Nowe zgłoszenia do backlogu
3. Tematy organizacyjne (jeśli były)
4. Decyzje ad-hoc (jeśli były)

## Przetwarzanie

Notatki z Daily są przetwarzane przez agenta `project-mapper` i mapowane na dokumentację projektów w katalogu `projekty/`.

**Rejestr przetworzonych:** `Notatki/_rejestr_przetworzonych.md`
